# 旧电脑（Win98 复古桌面网站）— 项目规划提示词

> 本文档是整个项目的「宪法」：既是规划书，也是以后每次迭代时可以直接喂给 AI 或新协作者的提示词。
> 任何新功能开发前，请先完整阅读本文档，遵守其中的约定。

---

## 1. 项目愿景

做一个**仿 Windows 98 桌面**的个人网站。整个网站就是一张「电脑桌面」：

- 桌面上的每一个图标 = 一个功能模块的入口（点击后打开一个 Win98 风格的窗口，运行我们自己写的功能），或者一个外部链接的入口（点击后跳转）。
- 网站的功能拓展 = **增删桌面图标**。加功能就是加图标、加窗口应用，不改框架。
- 同时适配 PC 端（鼠标、可拖拽缩放窗口）和手机端（触屏、窗口默认最大化）。

目标氛围：让人打开网站的瞬间以为穿越回了 1998 年。

本站定名「旧电脑」：复古的壳 × 现代的芯——机器会旧，人会老，想象力不会。主题定位、内容容器规划、作品档案规范见根目录 `THEME.md`（每次项目开启/阶段收尾必读，方向演进时迭代其版本号）。

## 2. 核心设计原则（不可违背）

1. **配置驱动**：桌面图标全部定义在 `js/config.js` 的 `WIN98_MODULES` 数组里。增删功能**只改这一个数组**（外加新模块的渲染函数），严禁把图标写死在 HTML/JS 其他位置。
2. **零构建、纯静态**：原生 HTML/CSS/JS，不用任何框架、不引入打包工具。双击 `index.html` 就能跑，扔到任何静态托管（GitHub Pages / Vercel / 服务器目录）就能上线。
   - **唯一例外**：`exhibits/` 展柜目录允许现代工具链（Vite + Vue + Tailwind）开发特效展品；构建产物（纯静态、无外链）提交进 git，主站以 iframe 开窗展示。主站本体永远零构建。注意展品页是 ES module，`file://` 双击打不开（http 服务与线上均正常）。
3. **离线自足**：所有美术素材（样式、字体、图标）一律下载到本地 `assets/` / `css/`，不允许依赖外链 CDN，保证网站永远能打开。
4. **小而美地迭代**：每次只加一两个模块，每个模块都要完整可用，不留半成品占位。
5. **还原度优先**：视觉细节向 Windows 98 靠拢（像素图标、凸起/凹陷边框、青色桌面、像素字体），新模块的 UI 必须看起来像「当年能跑在 Win98 上的程序」。

## 3. 技术选型

| 项 | 选择 | 原因 |
|---|---|---|
| 基础样式库 | [98.css](https://github.com/jdan/98.css) v0.1.20（已本地化到 `css/98.css`） | 最正宗的 Win98 组件样式（窗口、按钮、输入框、状态栏） |
| 字体 | 98.css 自带的 Pixelated MS Sans Serif（woff/woff2 已本地化），中文回退到系统中文字体 | 像素风英文 + 系统中文，兼顾还原度与可读性 |
| 图标素材 | **自绘像素图标**（`tools/make_icons.py` 用 Pillow 程序生成，已产出到 `assets/icons/`） | 零版权风险，风格统一，加新图标改脚本重跑即可 |
| 脚本 | 原生 JS，普通 `<script>` 标签按序加载，全局命名空间 `window.WIN98_*` | 不用 ES module，保证 `file://` 直接双击打开也能运行 |
| 数据持久化 | `localStorage` | 纯前端，够用 |

## 5. 架构设计

### 5.1 模块注册表 `js/config.js`

桌面上的每个图标是 `WIN98_MODULES` 数组里的一条记录：

```js
{
  id: 'notepad',                 // 唯一 ID
  title: '记事本',                // 图标下方显示的名字
  icon: 'assets/icons/notepad.png',
  type: 'window',                // 'window' = 打开内置功能窗口 | 'link' = 打开外部链接
  app: 'notepad',                // type=window 时：对应 WIN98_APPS 里的渲染函数名
  width: 520, height: 400,       // type=window 时：窗口初始尺寸（可选）
  url: 'https://...',            // type=link 时：目标地址（新标签页打开）
  showInStartMenu: true          // 是否同时出现在开始菜单（可选，默认 true）
}
```

### 5.2 应用注册表 `js/apps.js`

每个窗口型模块对应一个渲染函数：

```js
window.WIN98_APPS['myapp'] = function (bodyEl, win, moduleConfig) {
  // bodyEl：窗口内容区域（.window-body）的 DOM 元素，往里填内容即可
  // win：窗口管理器返回的窗口句柄（可调用 win.close() 等）
  // moduleConfig：config.js 里该模块的配置对象
};
```

### 5.3 运行时流程

1. `main.js` 启动 → `desktop.js` 读 `WIN98_MODULES` 渲染桌面图标 → `taskbar.js` 渲染开始菜单和时钟。
2. 用户双击图标：
   - `type: 'link'` → `window.open(url, '_blank')`；
   - `type: 'window'` → `windowManager.open(module)` 创建窗口，调用 `WIN98_APPS[module.app]` 填充内容。
3. `windowManager` 负责窗口生命周期：拖拽（Pointer Events，鼠标触屏通吃）、聚焦置顶、最小化到任务栏、最大化/还原、关闭；同一模块默认单例（重复打开只聚焦已有窗口）。
4. 任务栏按钮与窗口一一对应：点击切换聚焦/最小化。

### 5.4 响应式策略

- 断点：`640px`。
- PC 端：窗口按配置尺寸打开、可拖拽、可最大化；桌面图标从左上角按列排列。
- 手机端（`≤640px`）：
  - 窗口**默认最大化**打开（占满任务栏以上的区域），仍可拖动标题栏但基本无必要；
  - 桌面图标缩小、自动换行铺满宽度；
  - 所有可点目标（图标、按钮、开始菜单项）加大触控热区；
  - 禁用选中文字、长按弹出菜单等干扰操作的行为。

## 6. 如何新增一个桌面图标（迭代指南）

### 加一个外部链接入口（1 分钟）

只改 `js/config.js`，往 `WIN98_MODULES` 里加一条 `type: 'link'` 的记录即可。图标用 `tools/make_icons.py` 自绘生成（见下）。

### 加一个新的功能模块（标准动作）

1. 在 `tools/make_icons.py` 里为新模块画一个图标（仿照现有 `draw_xxx` 函数，32×32 像素画），注册到 `ICONS` 列表后运行 `python3 tools/make_icons.py`，PNG 自动产出到 `assets/icons/`。
2. 在 `js/apps.js` 里注册渲染函数 `WIN98_APPS['新模块id'] = function (bodyEl, win, cfg) { ... }`，UI 一律使用 98.css 的组件类（`button` / `field-row` / `sunken-panel` / `status-bar` / `tree-view` 等）。
3. 在 `js/config.js` 的 `WIN98_MODULES` 里加一条 `type: 'window'` 记录，`app` 指向上一步的函数名。
4. 打开 `index.html` 验证：桌面出现图标 → 双击打开窗口 → 手机上检查默认最大化是否正常。

> 如果某个模块代码量大（比如扫雷、画图），允许单独建 `js/apps/xxx.js` 文件，在 `index.html` 里用 `<script>` 引入，注册方式不变。

### 删除一个功能

从 `WIN98_MODULES` 删掉对应记录即可（桌面、开始菜单自动消失）；确认没有其他地方引用后，再删 `apps.js` 里的渲染函数。

### 给 AI 的迭代提示词模板

以后每次加功能，可以直接这样对 AI 说：

> 请阅读 `win98/PROJECT_PLAN.md` 并严格遵守其中的架构约定。现在新增一个桌面模块「XXX」，要求：（功能描述）。请按文档第 6 节的标准动作完成：用 `tools/make_icons.py` 生成图标、在 `js/apps.js` 注册渲染函数、在 `js/config.js` 注册图标，并在 PC/手机两种宽度下验证。

做 Backlog 时同样遵守第 2 节原则：配置驱动、小而美、不留半成品。

## 10. 本地运行与部署

- 本地预览：直接双击 `index.html`；或在 `win98/` 目录执行 `python3 -m http.server 8080` 后访问 <http://localhost:8080>。
- 部署：整个 `win98/` 目录原样上传到任意静态托管（GitHub Pages、Vercel、Netlify、Nginx 目录均可），无需任何构建步骤。
